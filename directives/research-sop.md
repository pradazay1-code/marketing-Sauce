# Research SOP

## When to Use
Invoked whenever the user needs deep research on industries, competitors, market trends, tools, strategies, or any topic relevant to the agency or clients.

## Research Types
1. **Industry Research** — Understanding a client's market
2. **Competitor Analysis** — What competitors are doing
3. **Market Trends** — What's happening in marketing/business
4. **Tool Research** — Evaluating new tools, platforms, software
5. **Client Research** — Deep dive on a specific prospect or client

## Step-by-Step Process

### Step 1: Define Research Scope
- What exactly do we need to learn?
- Why do we need it? (Decision to make, strategy to build, client to impress)
- What format should the output be in?
- What's the deadline?

### Step 2: Gather Information
- Search multiple sources
- Cross-reference data points
- Note conflicting information and assess credibility
- Look for primary sources over secondary

### Step 3: Analyze and Synthesize
- Identify key themes and patterns
- Separate facts from opinions
- Highlight actionable insights
- Note gaps in information

### Step 4: Document Findings
Structure research output as:
```
## Research: [Topic]
**Date:** [date]
**Purpose:** [why this research was needed]

### Key Findings
1. [Finding 1 with supporting detail]
2. [Finding 2 with supporting detail]
3. [Finding 3 with supporting detail]

### Actionable Insights
- [What to do based on finding 1]
- [What to do based on finding 2]

### Sources
- [Source 1]
- [Source 2]

### Recommendations
- [Recommendation based on all findings]
```

### Step 5: Save to Memory
- Save research to `memory/learnings.md` if broadly useful
- Save to client file if client-specific
- Tag with date and topic for easy retrieval

## Quality Checks
- [ ] Multiple sources consulted
- [ ] Facts are verifiable
- [ ] Insights are actionable (not just information dumps)
- [ ] Output is clear and well-organized
- [ ] Saved to appropriate memory file

## Notes
- Research should always lead to ACTION, not just information
- Keep research documents concise — focus on what matters
- Update research when new info becomes available
- Flag outdated research in memory files
