# Client Onboarding SOP

## When to Use
Invoked whenever a new client is signed and needs to be onboarded into the agency's system, or when the user says "onboard [client name]."

## Goals of Onboarding
1. Collect all necessary information from the client
2. Set clear expectations on deliverables and timelines
3. Get all access/credentials needed to do the work
4. Create the client's internal file in `memory/clients.md`
5. Set up their first deliverables

## Step-by-Step Process

### Step 1: Welcome & Kickoff
- Send welcome message/email (use template from `templates/`)
- Schedule kickoff call
- Send onboarding questionnaire

### Step 2: Collect Client Information
Gather and save to `memory/clients.md`:
- [ ] Business name
- [ ] Industry/niche
- [ ] Owner/decision-maker name and contact info
- [ ] Business website (current)
- [ ] Social media accounts
- [ ] Brand assets (logo, colors, fonts, brand guide)
- [ ] Target audience description
- [ ] Main competitors (3-5)
- [ ] Current marketing efforts (what they've tried, what's worked)
- [ ] Goals for working with us (specific, measurable)
- [ ] Budget range
- [ ] Services contracted (website, ads, content, etc.)
- [ ] Contract start/end dates
- [ ] Payment terms

### Step 3: Get Access & Credentials
- [ ] Google Analytics access
- [ ] Google Ads account access (if running ads)
- [ ] Meta Business Suite access (if running Meta ads)
- [ ] Social media account access (posting permissions)
- [ ] Website admin access (or hosting credentials)
- [ ] Domain registrar access (if managing domain)
- [ ] Any other platform-specific access needed
- **Store credentials securely — NEVER put in plain text files**

### Step 4: Competitive Analysis
- Research 3-5 competitors
- Analyze their websites, social media, ads, content
- Identify opportunities to differentiate
- Document findings in client file

### Step 5: Strategy Development
Based on client goals and research, create:
- [ ] 90-day marketing strategy outline
- [ ] Monthly deliverables checklist
- [ ] KPIs to track
- [ ] Reporting schedule (weekly/biweekly/monthly)
- Save strategy to client file

### Step 6: Set Up Project Infrastructure
- Create client section in `memory/clients.md`
- Set up reporting templates
- Create content calendar (if content is part of the scope)
- Set up ad accounts (if ads are part of the scope)
- Begin website build (if website is part of the scope)

### Step 7: First Deliverable
- Identify the quickest win to deliver in the first week
- This builds trust and shows value immediately
- Could be: website audit, first batch of content, ad account audit, etc.

### Step 8: Ongoing Communication
- Set up regular check-in schedule
- Define preferred communication channel (email, Slack, phone, etc.)
- Set expectations on response times
- Document all communication preferences in client file

## Onboarding Questionnaire
```
1. Tell us about your business in one sentence.
2. Who is your ideal customer? (Age, location, interests, income level)
3. What are your top 3 business goals for the next 6 months?
4. What marketing have you tried before? What worked and what didn't?
5. Who are your top 3-5 competitors?
6. What makes your business different from competitors?
7. Do you have brand guidelines? (Colors, fonts, logo files)
8. What's your monthly marketing budget?
9. How do you prefer to communicate? (Email, phone, Slack, etc.)
10. Anything else we should know about your business or goals?
```

## Quality Checks
- [ ] All client information collected and saved
- [ ] Access and credentials secured
- [ ] Strategy document created and approved
- [ ] First deliverable identified and in progress
- [ ] Communication preferences documented
- [ ] Client feels welcomed and confident in working with us

## Notes
- First impressions matter — make onboarding smooth and professional
- Deliver the first quick win within 7 days
- Set realistic expectations from day one
- Log any special requests or preferences in the client file
- Update this SOP based on feedback from new clients
